AWS Account & Free Tier Setup

Complete guide to set up your AWS account and use free tier services safely

Back to Home

Create AWS Free Tier Account

You will need a valid credit card. AWS won't charge without explicit action.
1 Visit AWS Sign Up Page

Go to aws.amazon.com/free

2 Create Account
  • Enter email address
  • Create a strong password
  • Enter AWS account name (your company)
  • Click Create
3 Verify Contact Information
  • Select "Personal" account type
  • Enter full name and address
  • Verify phone number with SMS
4 Add Payment Information

Enter credit card details. You won't be charged unless you exceed free tier limits.

AWS may charge $1 to verify your card - this will be refunded.
5 Verify Identity

Confirm via SMS or phone call

6 Choose Support Plan

Select "Free" for basic support (sufficient for learning)

Account created! You can now log in to AWS Management Console.

AWS CLI Installation & Configuration

1 Install AWS CLI

Windows: Download installer from AWS CLI MSI

macOS:

curl "https://awscli.amazonaws.com/awscli-macos.pkg" -o "AWSCLIV2.pkg" sudo installer -pkg AWSCLIV2.pkg -target /

Linux:

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip" unzip awscliv2.zip sudo ./aws/install
2 Verify Installation
aws --version

Should output: aws-cli/2.x.x Python/3.x.x ...

3 Create IAM User for AWS CLI
  • Log in to AWS Management Console
  • Search for "IAM" service
  • Click "Users" in left menu
  • Click "Create user"
  • Enter username (e.g., "dataeng-user")
  • Click "Next"
4 Attach Permissions Policy
  • Select "Attach policies directly"
  • Search for and select "AdministratorAccess" (for learning)
  • Click "Next" then "Create user"
Use restrictive policies in production. AdminAccess is only for learning.
5 Create Access Keys
  • Click on the user you created
  • Go to "Security credentials" tab
  • Click "Create access key"
  • Select "Command Line Interface (CLI)"
  • Click "Next" and confirm
  • Copy Access Key ID and Secret Access Key
Keep these keys secret! Never commit them to git.
6 Configure AWS CLI
aws configure

Follow the prompts:

  • AWS Access Key ID: [paste your access key]
  • AWS Secret Access Key: [paste your secret]
  • Default region: us-east-1
  • Default output format: json
7 Test Configuration
aws s3 ls

Should list your S3 buckets (empty if none created yet)

AWS CLI is configured and working!

Setting Up Billing Alerts

Critical: Set up billing alerts to avoid unexpected charges!
1 Enable Billing Alerts
  • Go to AWS Management Console
  • Click your account name (top right)
  • Click "Billing Dashboard"
  • In left menu, click "Billing preferences"
  • Check "Receive Free Tier Usage Alerts"
  • Check "Receive Billing Alerts"
  • Save preferences
2 Create Billing Alarm (Optional but Recommended)
  • Go to CloudWatch service
  • Click "Alarms" in left menu
  • Click "Create alarm"
  • Select metric: "Billing" > "Total Estimated Charge"
  • Set threshold (e.g., $10)
  • Create SNS topic for notifications
  • Confirm SNS subscription via email
Recommended threshold: $1-10 for free tier learning

AWS SDK (Boto3) Installation

1 Install Boto3
pip install boto3
2 Basic Python Example
import boto3 # Create S3 client s3_client = boto3.client('s3') # List S3 buckets response = s3_client.list_buckets() for bucket in response['Buckets']: print(f'Bucket: {bucket["Name"]}')
3 Example: Create S3 Bucket
import boto3 s3_client = boto3.client('s3') # Create bucket bucket_name = 'my-data-bucket-' + str(int(time.time())) s3_client.create_bucket(Bucket=bucket_name) print(f'Bucket {bucket_name} created!')
4 Example: Upload File to S3
import boto3 s3_client = boto3.client('s3') # Upload file s3_client.upload_file( 'local_file.txt', 'my-data-bucket', 'uploaded_file.txt' ) print('File uploaded!')
More examples: Boto3 Documentation

Cost Optimization Tips

1 Clean Up Unused Resources
  • Delete unused EC2 instances
  • Stop RDS databases when not in use
  • Delete old S3 buckets
  • Remove unused Elastic IPs
  • Delete old snapshots
2 Monitor Your Spending
  • Check Billing Dashboard daily
  • Use AWS Cost Explorer
  • Set up billing alerts
  • Enable free tier alerts
3 Use Cost Optimization Tools

AWS Trusted Advisor: Free recommendations for optimization

  • Log in to AWS Console
  • Search for "Trusted Advisor"
  • Review "Cost Optimization" recommendations
4 Best Practices
  • Use auto-scaling for variable workloads
  • Choose appropriate instance sizes
  • Enable S3 lifecycle policies
  • Use Spot Instances for non-critical tasks
  • Store infrequent data in Glacier
Always remember: Free tier is limited. Unexpected usage can result in charges. Monitor closely!

Security Best Practices

Never share AWS credentials. Use IAM roles and policies instead.
1 Root Account Security
  • Enable MFA (Multi-Factor Authentication) on root
  • Don't use root for daily tasks
  • Keep root credentials secure
  • Use IAM users instead
2 Enable MFA
  • Log in to AWS Console (root account)
  • Click account name â†’ My Account
  • Scroll to "Multi-factor authentication"
  • Activate MFA Device
  • Use authenticator app (Google Authenticator, Authy)
3 IAM Best Practices
  • Create IAM users for each person
  • Use IAM roles for services
  • Apply least privilege principle
  • Regularly rotate access keys
  • Never hardcode credentials in code
4 Secure Credential Storage

Never commit credentials to Git. Use environment variables or AWS Secrets Manager:

# Instead of hardcoding: # aws_access_key = "AKIAIOSFODNN7EXAMPLE" # Use environment variables: import os aws_access_key = os.environ.get('AWS_ACCESS_KEY_ID')

.gitignore template:

# AWS credentials .aws/ aws_credentials.py .env *.key *.pem
5 Enable Logging and Auditing
  • Enable CloudTrail for API logging
  • Enable S3 access logging
  • Enable RDS audit logs
  • Review logs regularly
6 Network Security
  • Use VPC for isolating resources
  • Configure Security Groups (firewalls)
  • Use Network ACLs for additional control
  • Block public access to S3 by default
  • Use VPC endpoints for private access
